# Document Status & Naming (_MASTER)

## Purpose
Define how documentation maturity, ownership, and authority are enforced across this repository.

## Status Definitions
- _MASTER  
  Operations-owned. Production authoritative. Complete and reliable.
- _INCOMPLETE  
  Project-owned. Usable reference. Not production-ready.
- _DRAFT  
  Project-owned. Intended design and partial facts.

## Lifecycle
_DRAFT → _INCOMPLETE → _MASTER

## Minimum Content Requirements
Every document must include:
- Purpose
- Scope
- Current known truth
- Intended final state (if evolving)

List-only, title-only, or placeholder-only documents are not permitted.
