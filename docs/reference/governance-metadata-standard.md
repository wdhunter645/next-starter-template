# Governance Metadata Normalization Standard

## Required Authority Headers

Authoritative documentation should standardize:

- Doc Type
- Audience
- Authority Level
- Owns
- Does Not Own
- Canonical Reference
- Last Reviewed

## Optional Extended Headers

Where applicable:

- Supersedes
- Superseded By
- Related Issues
- Related PRs
- Implementation Tracking

## Ownership Rules

A document may define only the authority it explicitly owns.

Documents must not silently redefine:

- runtime behavior
- routing
- schema ownership
- governance precedence
- implementation authority

unless explicitly designated as canonical authority.
