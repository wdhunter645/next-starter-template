# Program 5 — Ideas and Project Drafts

## Purpose

Provide a discussion inventory for Program 5.

Program 5 is the ideas and project-drafts lane. It contains items that are not yet portfolio-ready and therefore are not yet assigned to Programs 1-4.

This document is a discussion list only. It does not launch work, create issues, authorize Cursor execution, or establish final priority.

## Program 5 operating definition

Program 5 items need one or more of the following before promotion:

- clearer project definition;
- design documentation;
- scope boundaries;
- readiness review;
- dependency review;
- priority decision;
- placement into Programs 1-4;
- decomposition into project/task issues.

Legacy `#1379` currently represents the prior ideas/future-projects source until a dedicated Program 5 authority issue exists.

## Promotion checklist

Before an item leaves Program 5:

1. Bill or owner approves promotion review.
2. Atlas writes or identifies a repository design/readiness source.
3. The item is classified as project, task, or deferred idea.
4. Dependencies and non-interference risks are documented.
5. The item is prioritized against existing Programs 1-4 work.
6. The item is assigned to a Program 1-4 lane.
7. A source issue and implementation plan are created or authorized.
8. Cursor receives only bounded issue-level implementation instructions.
