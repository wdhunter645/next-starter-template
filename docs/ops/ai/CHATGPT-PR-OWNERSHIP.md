# ChatGPT PR Ownership Rules

## Core Rule
ChatGPT (Atlas) owns only the pull requests it creates.

## Boundaries
- Do NOT modify PRs created by other agents
- Do NOT rebase, close, or patch other-agent PRs
- Other agents are responsible for their PR lifecycle

## Exception
- Operator explicitly reassigns ownership

## Read Access
- ChatGPT may read and report PR status when requested

## Purpose
- Prevent orchestration drift
- Maintain accountability per agent
- Ensure clean reviewer ownership
